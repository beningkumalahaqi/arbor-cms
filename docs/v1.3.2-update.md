# v1.3.2 — Database-Backed File Storage

**Date:** March 6, 2026

---

## Summary

This update replaces the filesystem-based file storage with a database-backed implementation. Files and folders are now stored as binary blobs in the database using two new Prisma models (`StorageFile` and `StorageFolder`), making storage fully persistent across deployments on platforms with ephemeral filesystems like Heroku. A broken "Upload File" button in the file explorer was also fixed.

---

## Why

The previous implementation stored files on the local filesystem (`/storage` directory). This breaks on production platforms with ephemeral filesystems — files are lost on every deploy, restart, or dyno cycle. The new approach stores everything in the same database, requiring no external storage service.

---

## Changes

### 1. New Database Models

**File:** `prisma/schema.prisma`

Two new models added:

- **`StorageFile`** — Stores uploaded files as binary blobs with `id`, `name`, `path` (unique), `mimeType`, `size`, `data` (Bytes), `createdAt`, `updatedAt`. Indexed on `path`.
- **`StorageFolder`** — Represents virtual folder structure with `id`, `path` (unique), `createdAt`. Indexed on `path`.

**File:** `prisma/migrations/20260306000000_add_storage_models/migration.sql`

Migration SQL that creates both tables with unique constraints and indexes.

### 2. Storage Module Restructured

The single `lib/storage.ts` file was replaced with a `lib/storage/` module:

| File | Purpose |
|------|---------|
| `lib/storage/types.ts` | Shared `FileEntry` interface, name sanitization, MIME type mapping, `isImageFile` helper |
| `lib/storage/db.ts` | Database-backed implementation — all CRUD operations use Prisma queries on `StorageFile`/`StorageFolder` |
| `lib/storage/index.ts` | Re-exports everything from `db.ts` and `types.ts` |

All existing function signatures are preserved — no changes needed in consuming code:
- `ensureStorageRoot()` — now a no-op (database needs no initialization)
- `listDirectory()` — queries files and folders, filters to direct children
- `listDirectoryTree()` — recursively builds tree
- `createFolder()` — upserts a `StorageFolder` record
- `saveFile()` — upserts a `StorageFile` with binary data
- `deleteEntry()` — deletes file or folder and all children
- `renameEntry()` — updates paths for file/folder and all descendants
- `moveEntry()` — moves file/folder to new parent, updating all descendant paths
- `readFile()` — fetches binary data from `StorageFile`
- `ensurePageFolder()` — upserts a `StorageFolder` for page paths

Path traversal prevention is maintained via `normalizePath()` which rejects `..` segments.

### 3. Upload File Button Fix

**File:** `components/admin/file-explorer.tsx`

The "Upload File" button had `pointer-events-none` on the inner `<Button>` component, which prevented click events from reaching the hidden `<input type="file">` through the `<label>`. Replaced with a styled `<label>` element that matches the Button `sm`/`secondary` variant styles, allowing clicks to properly trigger the file picker.

### 4. API Route Cleanup

**File:** `app/api/storage/route.ts`

Updated imports to use the new module path. The `ensureStorageRoot()` calls are retained (they're a no-op in the DB backend but keep the interface consistent).

### 5. Documentation Updates

- **`README.md`** — Updated File Management section to describe database-backed storage, updated tech stack, updated project structure
- **`.github/copilot-instructions.md`** — Updated folder structure, added `StorageFile`/`StorageFolder` model docs, updated API route docs, added filesystem storage prohibition to "What NOT to do"

## Behavior Summary

| Feature | Behavior |
|---------|----------|
| File uploads | Stored as binary blobs in `StorageFile` table |
| Folder creation | Stored as path records in `StorageFolder` table |
| File serving | Read from database via `/api/storage/file/[...path]` with cache headers |
| Page folder auto-creation | Creates `StorageFolder` record on page creation |
| Upload button | Now clickable — styled label triggers hidden file input |
| Path traversal | Prevented via `normalizePath()` rejecting `..` segments |
| Max file size | 10MB limit enforced in API route |
| Blocked extensions | `.exe`, `.bat`, `.sh`, `.js`, `.php`, etc. still blocked |
