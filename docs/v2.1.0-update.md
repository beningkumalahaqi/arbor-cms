# v2.1.0 ‚Äî Environment Sync

**Date:** March 13, 2026

---

## Summary

This update adds an **Environment Sync** feature to Arbor CMS. Administrators can now synchronize content between the current CMS database and a target environment database ‚Äî enabling content migration between staging, production, or any other deployment. The feature includes a dedicated admin page for running sync operations, a settings page for configuring target database credentials, and a sync service module that handles safe, upsert-based content transfer.

---

## Changes

### 1. Database Schema ‚Äî SiteSettings Extension

**File:** `prisma/schema.prisma`

**New fields on `SiteSettings`:**
- `environmentDatabaseUrl` (string, default "") ‚Äî LibSQL/Turso URL for the target environment database
- `environmentDatabaseToken` (string, default "") ‚Äî Optional authentication token for the target database

These fields store the target environment connection details. They are excluded from sync operations to prevent credentials from being copied between environments.

### 2. Sync Service Module

**File:** `lib/sync/service.ts`

A dedicated service module that handles all synchronization logic. Exported as `syncService` with three methods:

| Method | Purpose |
|--------|---------|
| `testConnection(url, authToken)` | Tests connectivity to a target database, returns `{ connected, error? }` |
| `syncToTarget(source, targetUrl, targetToken)` | Copies all CMS content from the current environment to the target |
| `syncFromTarget(source, targetUrl, targetToken)` | Copies all CMS content from the target environment to the current |

**Sync order (dependency-safe):**
1. **SiteSettings** ‚Äî Navigation and footer configuration (excludes environment sync credentials)
2. **PageTypeSettings** ‚Äî Page type icons and allowed children
3. **Pages** ‚Äî Ordered by `fullPath` ascending to sync parent pages before children
4. **Widgets** ‚Äî Parent widgets first, then child widgets (respects container nesting)
5. **FormTypes** ‚Äî Form structure definitions
6. **FormSubmissions** ‚Äî Individual form responses (skips on referential integrity failure)
7. **StorageFolders** ‚Äî Virtual folder structure
8. **StorageFiles** ‚Äî Uploaded file data as binary blobs

**Key design decisions:**
- All entities use **upsert by ID** ‚Äî creates if new, updates if existing, avoids duplicates
- **URL validation** ‚Äî only `libsql://`, `https://`, and `file:` protocols are allowed
- **Client lifecycle** ‚Äî target Prisma client is created per operation and disconnected in a `finally` block
- **Error isolation** ‚Äî form submission sync failures are logged but don't block the overall sync

**Return type ‚Äî `SyncResult`:**
```typescript
{
  success: boolean;
  message: string;
  details?: {
    pages: number;
    widgets: number;
    pageTypeSettings: number;
    formTypes: number;
    formSubmissions: number;
    siteSettings: boolean;
    storageFiles: number;
    storageFolders: number;
  };
}
```

### 3. API Routes

**New ‚Äî `POST /api/environment-sync`:**
- Authentication required
- Accepts `{ direction: "to" | "from" }`
- Loads target URL and token from `SiteSettings`
- Calls `syncToTarget()` or `syncFromTarget()` accordingly
- Returns `SyncResult` with 200 (success) or 500 (failure) status

**New ‚Äî `GET /api/environment-sync/status`:**
- Authentication required
- Returns connection status for both environments:
  - **Current environment** ‚Äî always connected (the running CMS database)
  - **Target environment** ‚Äî tests connectivity using `syncService.testConnection()`, returns status and any error message
- Target URL is masked in the response for security

**Updated ‚Äî `GET /api/site-settings`:**
- Now returns `environmentDatabaseUrl` and `environmentDatabaseToken` fields

**Updated ‚Äî `PUT /api/site-settings`:**
- Now accepts `environmentDatabaseUrl` and `environmentDatabaseToken` for persistence

### 4. Admin Page ‚Äî Environment Sync

**File:** `app/admin/environment-sync/page.tsx`

**Route:** `/admin/environment-sync`

A dedicated admin page for viewing environment status and running sync operations:

**Status cards (2):**
- **Current Environment** ‚Äî Shows connection status and masked database URL
- **Target Environment** ‚Äî Shows connection status, masked URL, and any error message

**Sync actions:**
- **"Sync to Target Environment"** button ‚Äî Copies content from current ‚Üí target
- **"Sync from Target Environment"** button ‚Äî Copies content from target ‚Üí current
- Both buttons are disabled when the target is not connected
- A helper message links to the settings page when no target is configured

**Safety features:**
- **Confirmation dialog** ‚Äî warns about overwriting content before each sync
- **Loading state** ‚Äî buttons show "Syncing..." while operation is in progress
- **Result card** ‚Äî displays success/failure badge, message, and a grid of entity counts after sync completes

**URL masking:** All database URLs are masked in the UI (e.g., `libsql://abcd****efgh`) to protect sensitive connection details.

### 5. Settings Page ‚Äî Environment Sync Configuration

**File:** `app/admin/settings/environment-sync/page.tsx`

**Route:** `/admin/settings/environment-sync`

A settings sub-page for configuring the target database connection:

**Fields:**
- **Environment Database URL** ‚Äî text input with placeholder `libsql://your-database-url.turso.io`
- **Environment Database Token** ‚Äî password input (masked) for the authentication token
- **Save** button ‚Äî persists values via `PUT /api/site-settings`
- Success message displayed for 3 seconds after saving

### 6. Settings Index ‚Äî Environment Sync Link

**File:** `app/admin/settings/page.tsx`

A new card added to the main settings page below the existing Theme, Navigation, and Footer sections:

- **Title:** "Environment Sync" with a sync icon
- **Description:** "Configure a target environment database to enable content synchronization between environments."
- Links to `/admin/settings/environment-sync`
- Styled as a hoverable card matching the existing settings pattern

### 7. Admin Shell ‚Äî Navigation Entry

**File:** `components/admin/admin-shell.tsx`

A new sidebar navigation item added between "Forms" and "Settings":

- **Label:** "Environment Sync"
- **Route:** `/admin/environment-sync`
- **Icon:** Sync arrows SVG icon (polyline-based refresh/sync pattern)
