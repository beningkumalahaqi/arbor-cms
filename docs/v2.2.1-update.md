# v2.2.1 — Environment Sync Widget FK Fix

**Date:** March 25, 2026

---

## Summary

This patch fixes an Environment Sync failure that could occur while applying widgets on the target environment:

`Foreign key constraint violated on the foreign key`

The failure happened in `target.widget.upsert()` when a widget referenced a missing or not-yet-synced parent widget (`parentId`) or a missing page (`pageId`).

---

## Error Details

### Reported Error

```text
Sync failed: Invalid `target.widget.upsert()` invocation
...
for (const widget of payload.widgets) {
  await target.widget.upsert(...)
}
Foreign key constraint violated on the foreign key
```

### Why It Happened

The previous implementation processed widgets in a single pass. Even though payload ordering attempted to place parent widgets first, FK failures were still possible in inconsistent or partial datasets:

- A child widget could be processed before its parent existed on the target.
- A widget could reference a `pageId` that does not exist on the target.
- A widget could reference a `parentId` that no longer exists (orphaned relationship).

Because `Widget` has FK constraints to both `Page` (`pageId`) and `Widget` (`parentId`), the upsert failed and aborted sync.

---

## Solution

**File:** `lib/sync/service.ts`

Widget sync in `applySyncPayload()` was rewritten to be dependency-safe:

1. Load existing target `Page` IDs and existing/synced `Widget` IDs.
2. Sync root widgets first (`parentId = null`).
3. Sync child widgets only when their parent ID is already known on target.
4. Skip widgets whose `pageId` does not exist on target (with warning log).
5. Fallback for unresolved parents: sync widget as root (`parentId: null`) with warning instead of crashing.

This preserves forward progress for sync operations and prevents a single broken relationship from failing the entire environment sync.

---

## Behavior After Fix

- Environment sync no longer fails on `widget.upsert()` due to parent-ordering/orphan FK issues.
- Inconsistent widget relationships are handled defensively with warnings.
- Valid widgets continue syncing successfully.
