# v1.4.0 — shadcn/ui Migration & Dual-Theme System

**Date:** March 6, 2026

---

## Summary

This update migrates the entire UI component layer to shadcn/ui with a lime-green theme, replaces all hardcoded color classes with CSS variable design tokens, and introduces a dual-theme system that lets administrators control the CMS admin and live site appearances independently.

---

## Changes

### 1. shadcn/ui Component Library

**Files:** `components.json`, `lib/utils.ts`, `components/ui/*.tsx`

The project now uses [shadcn/ui](https://ui.shadcn.com) as its component library, configured with:
- **Style:** `new-york`
- **Base color:** `neutral`
- **Icon library:** `phosphor` (`@phosphor-icons/react`)
- **CSS variables:** enabled (oklch color space)

New shadcn components installed:
- `Badge`, `Label`, `Separator`, `Dialog`, `RadioGroup`, `Tooltip`

Existing components replaced with shadcn versions:
- `Button`, `Input`, `Textarea`, `Select`, `Card`, `Table`

The `Select` component is now a compound Radix component (`Select > SelectTrigger > SelectValue + SelectContent > SelectItem`). The `Card` component uses `CardHeader`, `CardContent`, `CardFooter` sub-components.

A `cn()` utility (`lib/utils.ts`) is provided for conditional class merging via `clsx` + `tailwind-merge`.

### 2. Lime Theme with CSS Variable Design Tokens

**File:** `app/globals.css`

All colors are defined as CSS custom properties using oklch color space. The lime-green primary color:
- Light: `oklch(0.532 0.157 131.589)`
- Dark: `oklch(0.841 0.238 116.029)`

Every component across the site now uses semantic design tokens instead of hardcoded Tailwind color classes:

| Old (hardcoded) | New (design token) |
|---|---|
| `text-zinc-900 dark:text-zinc-100` | `text-foreground` |
| `text-zinc-500 dark:text-zinc-400` | `text-muted-foreground` |
| `bg-white dark:bg-zinc-950` | `bg-background` |
| `bg-zinc-100 dark:bg-zinc-800` | `bg-muted` |
| `border-zinc-200 dark:border-zinc-800` | `border` |
| `text-red-500` | `text-destructive` |
| `text-blue-600 dark:text-blue-400` | `text-primary` |
| `bg-emerald-100 text-emerald-700` | `Badge variant="default"` |
| `bg-amber-100 text-amber-700` | `Badge variant="outline"` |

### 3. Inter Font

**File:** `app/layout.tsx`

The default font is now Inter (via `next/font/google`), replacing Geist Sans. Applied via `--font-inter` CSS variable and `font-sans` class on the body element.

### 4. Dual-Theme System

**File:** `components/theme-provider.tsx`

A new `ThemeProvider` context supports independent themes for the admin interface and the live site:

- **`adminTheme`** — Applies to `/admin/*`, `/login`, `/setup` routes
- **`siteTheme`** — Applies to all other (public) routes
- Both support `"auto"` | `"dark"` | `"light"` with `"auto"` as default
- Stored in `localStorage` under keys `theme-admin` and `theme-site`
- System preference changes are tracked when either theme is set to `"auto"`

**Flash prevention:** An inline `<script>` in `app/layout.tsx` reads the correct localStorage key based on the initial pathname before React hydrates, preventing a flash of incorrect theme on page load.

### 5. Settings Page

**File:** `app/admin/settings/page.tsx`

A new Settings page at `/admin/settings` with a Theme section containing two 3-option sliders:
- **CMS Admin** — Auto / Light / Dark
- **Live Site** — Auto / Light / Dark

The settings page is designed to be extensible for future settings sections. A "Settings" entry with a gear icon has been added to the admin sidebar navigation.

## Migration Notes

### For template authors
- Replace all `zinc-*`, `blue-*`, `emerald-*`, `amber-*` color classes with design tokens
- Use `text-foreground` instead of `text-zinc-900 dark:text-zinc-100`
- Use `text-muted-foreground` instead of `text-zinc-500 dark:text-zinc-400`
- Use `text-primary` instead of `text-blue-600 dark:text-blue-400`
- Remove explicit `dark:` variant prefixes for colors (handled by CSS variables)
- `dark:prose-invert` is still needed for Tailwind Typography

### For component consumers
- `Button`: `variant="danger"` → `variant="destructive"`
- `Select`: now a compound component — use `SelectTrigger`, `SelectValue`, `SelectContent`, `SelectItem`
- `Card`: wrap content in `CardHeader`/`CardContent` sub-components
- Import `cn` from `@/lib/utils` for conditional class merging
