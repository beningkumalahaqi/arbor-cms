# v2.2.2 - Dashboard Customization and Settings Refinement

**Date:** March 25, 2026

---

## Summary

This release improves the admin dashboard and settings UX without changing the core architecture.

Highlights:

- Configurable dashboard quick access cards
- Drag-and-drop card reordering with persistence
- Favorites/pinned modules prioritized at the top
- Recent activity section on dashboard
- Cleaner settings page grouping (Appearance and General Settings)

---

## Dashboard Improvements

### Quick Access Cards

The dashboard now includes a **Quick Access** section below the existing stats cards. Cards link to major admin modules:

- Pages
- Media / Files
- Page Types
- Forms
- Environment Sync
- Settings

Each card includes an icon, module title, short description, and direct navigation link.

### Drag-and-Drop Reordering

Quick access cards can now be reordered directly from the dashboard using native HTML drag-and-drop.

Behavior:

- Drag a card and drop it over another card
- New order is persisted via existing `PUT /api/site-settings`
- Saved order is restored when the dashboard reloads

### Favorites / Pinned Modules

Cards now support a star toggle to mark modules as favorites.

Behavior:

- Starred modules are treated as pinned
- Pinned modules always render before non-pinned modules
- Favorite state is persisted in settings

### Recent Activity

A new **Recent Activity** section is shown below quick access cards.

Data source:

- Derived from existing `Page` records, ordered by `updatedAt` descending
- Shows up to 8 recent items

Each item displays:

- Title (`name` fallback to `slug`)
- Activity type (`Created page` or `Updated page`)
- Page type
- Timestamp

---

## Settings Page Refinement

The settings page is now grouped into two clear sections:

### Appearance

- Theme
- Layout
  - Dashboard module visibility (show/hide)
  - Dashboard favorite/pinned state

### General Settings

- Navigation
- Footer
- Environment Sync

The page continues to use existing card patterns and autosave behavior.

---

## Persistence Strategy

No new storage subsystem was introduced.

Dashboard configuration is stored in existing `SiteSettings` as a JSON string field:

- `dashboardModules` (module id, enabled state, favorite state, and order)

This is read/written through the existing `/api/site-settings` route.
