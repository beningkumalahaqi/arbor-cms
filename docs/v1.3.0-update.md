# v1.3.0 — File Management, Image Properties, Article Page Type & Rich Text Editor

Released: March 5, 2026

---

## Summary

This release adds a complete file management system, a new `image` property type with a visual selector, the `article` page type, and a fully-featured WYSIWYG rich text editor powered by TipTap.

---

## New Features

### File Management System

A built-in file manager is now available in the admin panel under **Files**. It provides full CRUD operations for files and folders stored in the `/storage` directory.

**What's included:**
- **File Explorer UI** (`/admin/files`) — Browse, navigate, upload, rename, move, and delete files and folders
- **Drag-and-drop move** — Drag any file or folder onto a directory to move it there; drop onto the `..` row to move to the parent folder. Visual highlight on valid drop targets
- **Sortable columns** — Click column headers (Name, Size, Date Added, Modified) to sort entries. Click again to toggle ascending/descending. Arrow indicators (↑/↓) show the active sort direction; directories always appear first regardless of sort
- **Hierarchical folder navigation** with breadcrumbs and back button
- **File upload** — Any file type can be uploaded (max 10MB), with security blocking for executable extensions (.exe, .bat, .sh, .js, .php, etc.)
- **File serving API** — Files are served via `/api/storage/file/[...path]` with proper MIME types and caching headers
- **Auto-folder creation** — When a page is created, a corresponding folder is automatically created in `/storage/pages/` matching the page hierarchy (e.g., creating `/about/team` creates `storage/pages/about/team/`)
- **File name sanitization** — Only alphanumeric characters, hyphens, underscores, dots, and spaces are allowed
- **Path traversal prevention** — All storage operations are validated to stay within the storage root

**New files:**
- `lib/storage.ts` — Storage utility with CRUD operations, move, path safety, MIME types
- `app/api/storage/route.ts` — File management API (GET list, POST create/upload/rename/move, DELETE)
- `app/api/storage/file/[...path]/route.ts` — File serving endpoint
- `app/admin/files/page.tsx` — Admin file manager page
- `components/admin/file-explorer.tsx` — File explorer UI component with drag-and-drop and sortable columns

**Navigation:**
- "Files" link added to the admin sidebar navigation between "Pages" and "Page Types"

**API:**
- `POST /api/storage` with `action: "move"` — Moves a file or folder to a new destination directory. Accepts `sourcePath` and `destFolder`. Validates path safety, prevents moving a folder into itself, and checks for name conflicts at the destination.

### Image Property Type

A new `image` property type joins the existing `text` and `richText` types. Image properties store a reference to a file path in `/storage`.

**What's included:**
- `image` added to the `PropertyType` union in `lib/page-types/types.ts`
- **ImageField component** — Displays a preview of the selected image, with "Select Image" / "Change Image" / "Remove" buttons
- **ImageSelectorModal** — A visual modal that browses `/storage` and shows only image files in a grid layout. Supports folder navigation, image uploads directly from the modal, and selection confirmation
- Admin forms (`new` and `edit` pages) automatically render the `ImageField` for `image` type properties

**New files:**
- `components/admin/image-field.tsx` — Image field component with preview
- `components/admin/image-selector-modal.tsx` — Visual image selector modal

### Article Page Type

A new `article` page type is now available, designed for rich content pages with visual headers.

**Properties:**
| Property | Type | Required | Description |
|----------|------|----------|-------------|
| Title | `text` | Yes | Article headline |
| Image Banner | `image` | No | Full-width banner image |
| Content | `richText` | No | Article body content |

**Template layout:**
The article template renders a clean, readable layout:
1. Title — Large, bold heading
2. Image Banner — Full-width rounded image below the title
3. Content — Prose-formatted rich text with proper typography

**New files:**
- `lib/page-types/article.ts` — Article page type definition
- `lib/page-template/article-template.tsx` — Article page template

**Registry updates:**
- Registered in `lib/page-types/registry.ts`
- Template registered in `lib/page-template/registry.ts`

### Rich Text Editor (TipTap)

The `richText` property type has been upgraded from a plain `<textarea>` to a full WYSIWYG editor powered by TipTap (ProseMirror).

**Editor features:**
- **Text formatting** — Bold, italic, underline, strikethrough
- **Headings** — H1 through H4 via dropdown selector
- **Font size** — 12px to 32px via dropdown
- **Text color** — Color picker for text color
- **Lists** — Bullet lists and numbered/ordered lists
- **Text alignment** — Left, center, right alignment
- **Block elements** — Blockquote, code block, horizontal rule
- **Raw HTML mode** — Toggle between WYSIWYG editor and raw HTML textarea
- **Mode sync** — Changes in either mode are synced to the other when switching

**Dependencies added:**
- `@tiptap/react`, `@tiptap/starter-kit`, `@tiptap/pm`
- `@tiptap/extension-underline`, `@tiptap/extension-text-align`
- `@tiptap/extension-text-style`, `@tiptap/extension-color`
- `@tiptap/extension-image`
- `@tailwindcss/typography` — For `prose` class support in content rendering

**New file:**
- `components/admin/rich-text-editor.tsx` — Full editor component

**Updated templates:**
- `home-template.tsx`, `content-template.tsx`, `article-template.tsx` — Now use `dangerouslySetInnerHTML` with `prose` classes to render rich text HTML properly

- `FileEntry` interface now includes `createdAt` (birth time) for both files and directories, in addition to the existing `modifiedAt`

- New `moveEntry(sourcePath, destFolderPath)` function in `lib/storage.ts` — moves files and folders with path traversal prevention, self-move detection, and duplicate name checks

## Migration Notes

- No database schema changes — this release is fully backward-compatible
- The `/storage` directory is created automatically on first API call
- Existing `richText` content stored as plain text will render as-is in the new editor (wrapped in paragraph tags)
- Run `npm install` to install new TipTap dependencies
