# v1.1.0 — Home Page Singleton & Routing Enforcement

**Date:** March 5, 2026

---

## Summary

This update enforces the Home page type as a singleton that always serves the root URL `/`. Previously, a Home page could be created with any slug and nested under other pages like a regular content page. This update corrects that behavior at both the API and UI levels.

---

## Changes

### 1. Home Page Type — Singleton Enforcement (API)

**File:** `app/api/pages/route.ts`

The `POST` handler for page creation now includes three additional checks when `pageType` is `"home"`:

- **Singleton check:** Queries `prisma.page.findFirst({ where: { pageType: "home" } })`. If a home page already exists, the request is rejected with a `409` status and the message: *"A home page already exists. Only one home page is allowed."*
- **No parent allowed:** If `parentId` is provided for a home page, the request is rejected with a `400` status. The home page must always be a root-level page.
- **Forced root path:** When the page type is `home`, `fullPath` is always set to `"/"` regardless of any slug provided. The `slug` field is stored as an empty string.

Slug validation is skipped for the `home` type since it doesn't use a slug.

### 2. Admin UI — Conditional Form Fields

**File:** `app/admin/pages/new/page.tsx`

When the user selects "Home" as the page type in the new page form:

- The **Slug** and **Parent Page** fields are hidden.
- An informational message is displayed: *"Home page always serves /. Slug and parent are set automatically."*
- The internal form state resets `slug` to `""` and `parentId` to `""` via a `useEffect` hook.

For all other page types, the Slug and Parent Page fields render as before.

---

## Behavior Summary

| Rule | Enforcement |
|------|------------|
| Only one Home page can exist | Server-side singleton check before insert |
| Home page always serves `/` | `fullPath` forced to `"/"` in API |
| Home page cannot have a parent | `parentId` rejected server-side |
| Slug not required for Home | Validation skipped, stored as `""` |
| Admin form adapts to page type | Slug/parent fields hidden for Home |

---

## Impact

- No database schema changes.
- No migration required.
- Existing home pages created before this update with a non-root path (e.g., `/home`) should be manually deleted and recreated via the admin UI to get the correct `fullPath = "/"`.
